# DSCT & MID 实验现象与核心问题分析报告

## 1. 观察到的实验现象

在目前的硬路由（Hard Routing）架构与 DSCT/MID 实验中，主要观察到以下反直觉现象：

1. **`tag_routing` 基线的反常优势**：
   在混合了 `en, yo, so, ha` 四种语言并使用 的 `tag_routing` 模型中，其英文的 `truthfulqa_mc1` 得分达到了 **0.3754**。
   
2. **纯英文 SFT (`train_en`) 存在“知识遗忘”**：
   作为 Donor 共享底座的纯英文 SFT 模型，其 `truthfulqa_mc1` 得分仅为 **0.368**。这说明纯英文指令微调发生了一定程度的过拟合，导致了预训练阶段世界知识的灾难性遗忘。而 `tag_routing` 依靠多语言混合数据的正则化作用，反而保住了这部分知识。

3. **特化模型 (`mid_yo`, `dsct_ha` 等) 英语能力严重崩溃**：
   当我们把纯英文 Adapter (`train_en`) 和小语种 Adapter (`mid_yo`) 共同挂载测试时（不加 Tag），其英文得分暴跌至 **0.339**（如 `mid_yo` 的 `english_avg`）。这说明小语种 Adapter 严重污染了英文的处理通道，一旦硬路由发生误判，系统将面临灾难性的性能崩塌。

---

## 2. 核心问题剖析

为什么理论上应该实现“参数隔离”和“特征约束”的 DSCT 与 MID 机制失效了？代码实现中存在两个致命的 Bug，以及一个架构设计上的先天瓶颈：

### 问题一：MID 蒸馏损失的缩放 Bug（后天约束失效）
在 `train_mid.py` 和 `train_dsct.py` 中，计算 `_mid_loss` 时，将所有 Layer、所有 Token 上的余弦距离累加后，**错误地除以了总项数 `n`**（`n = batch_size * layers * tokens`）。
- **后果**：原本设定的超参数 $\alpha$ 和 $\beta$ 被稀释了数十倍甚至上百倍。模型在学习小语种时，受到的“不要忘记 Teacher 英文表征”的惩罚力度极小，几乎为 0。因此，模型肆无忌惮地修改了全局表征，导致英语能力暴跌至 0.339。

### 问题二：正交损失 (Ortho Loss) 的数学定义错误（后天隔离失效）
在 `train_dsct.py` 中，计算子空间正交性时，代码将矩阵展平为一维向量，并计算了余弦相似度的平方 (`v1 * v2 / (norm * norm)`)。
- **后果**：在高维空间中，两个随机初始化的展平向量天然就几乎正交（余弦相似度极近于 0）。这导致 Ortho Loss 在训练初期就已经是 0，根本没有产生梯度去强迫 $A_{spec}$ 和 $A_{donor}$ 的列空间分离。参数空间依然发生了重叠和覆写。

### 问题三：Teacher/Donor 的“先天不足”（天花板太低）
目前的架构将 `train_en` (得分 0.368) 作为 Donor/Teacher 传给了 DSCT 和 MID。
- **后果**：即使修复了上述两个 Bug，约束机制 100% 完美生效，`mid_yo` 的英语得分最高也只能被约束到 0.368。它在物理上不可能超越 `tag_routing` 的 0.375 天花板。这就导致了您认为的“硬路由无法超越 tag_routing”。

---

## 3. 破局方案与行动指南

为了实现完美的硬路由系统（即：小语种极强，且各个分支的英语事实性兜底都能达到 0.375+），建议执行以下两步：

### Action 1: 修复代码中的约束机制 Bug
1. **修复 `_mid_loss`**：将除以 `n` 修改为除以有效的 Batch 数量 (`valid_b`)，在样本内部求和，确保 $\alpha, \beta$ 惩罚力度生效。
2. **修复 `_ortho_loss`**：废弃展平向量的余弦计算，改用真正的矩阵子空间正交性公式：$\frac{||A_{don} A_{spec}^T||_F^2}{||A_{don}||_F^2 ||A_{spec}||_F^2}$。

### Action 2: “偷天换日”更换 Teacher/Donor
在重新启动 `train_mid.py` 或 `train_dsct.py` 时，**不再使用 `train_en` 作为 Donor Adapter，而是直接将 `tag_routing` 训练出的 Adapter 作为 Teacher 传入。**
- 这样，特化分支不仅能学到优秀的小语种能力，还会被强制约束去对齐 `tag_routing` 那高达 0.375 的高维英语事实性表征。
- 最终得到的硬路由各个分支，既有专精，又有顶级底蕴。
